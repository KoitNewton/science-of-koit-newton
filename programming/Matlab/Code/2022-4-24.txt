help gradient
 <strong>gradient</strong> Approximate gradient.
    [FX,FY] = <strong>gradient</strong>(F) returns the numerical gradient of the matrix F.
    FX corresponds to dF/dx, the differences in x (horizontal) direction.
    FY corresponds to dF/dy, the differences in y (vertical) direction.
    The spacing between points in each direction is assumed to be one.
    When F is a vector, DF = <strong>gradient</strong>(F) is the 1-D gradient.
 
    [FX,FY] = <strong>gradient</strong>(F,H), where H is a scalar, uses H as the constant
    spacing between points in each direction.
 
    [FX,FY] = <strong>gradient</strong>(F,HX,HY), when F is 2-D, uses the spacing specified
    by HX and HY. HX and HY can either be scalars to specify the spacing
    between coordinates or vectors to specify the coordinates of the
    points. If HX and HY are vectors, their lengths must match the
    corresponding dimension of F.
 
    [FX,FY,FZ] = <strong>gradient</strong>(F), when F is a 3-D array, returns the numerical
    gradient of F. FZ corresponds to dF/dz, the differences in the z
    direction. <strong>gradient</strong>(F,H), where H is a scalar, uses H as the constant
    spacing between points in each direction.
 
    [FX,FY,FZ] = <strong>gradient</strong>(F,HX,HY,HZ) uses the spacing given by HX, HY, HZ.
 
    [FX,FY,FZ,...] = <strong>gradient</strong>(F,...) extends similarly when F is N-D and
    must be invoked with N outputs and either 2 or N+1 inputs.
 
    Note: The first output FX is always the gradient along the first
    dimension of F, going across columns.  The second output FY is always
    the gradient along the second dimension of F, going across rows.  For
    the third output FZ and the outputs that follow, the Nth output is the
    gradient along the Nth dimension of F.
 
    Examples:
        x = -2:.2:2; y = (-1:.15:1).';
        z = x .* exp(-x.^2 - y.^2);
        [px,py] = gradient(z,.2,.2);
        contour(x,y,z), hold on
        quiver(x,y,px,py), hold off
 
    Class support for input F:
       float: double, single
 
    See also <a href="matlab:help diff">diff</a>, <a href="matlab:help del2">del2</a>.

help ones
 <strong>ones</strong>   Ones array.
    <strong>ones</strong>(N) is an N-by-N matrix of ones.
 
    <strong>ones</strong>(M,N) or <strong>ones</strong>([M,N]) is an M-by-N matrix of ones.
 
    <strong>ones</strong>(M,N,P,...) or <strong>ones</strong>([M N P ...]) is an M-by-N-by-P-by-... array of
    ones.
 
    <strong>ones</strong>(SIZE(A)) is the same size as A and all ones.
 
    <strong>ones</strong> with no arguments is the scalar 1.
 
    <strong>ones</strong>(..., CLASSNAME) is an array of ones of class specified by the 
    string CLASSNAME.
 
    <strong>ones</strong>(..., 'like', Y) is an array of ones with the same data type, sparsity,
    and complexity (real or complex) as the numeric variable Y.
 
    Note: The size inputs M, N, and P... should be nonnegative integers. 
    Negative integers are treated as 0.
 
    Example:
       x = ones(2,3,'int8');
 
    See also <a href="matlab:help eye">eye</a>, <a href="matlab:help zeros">zeros</a>.

    <strong>movie</strong>(M) plays the movie in matrix M once, using the current axes
    as the default target. To play the movie in the figure instead of
    the axes, specify the figure handle (or gcf) as the first

    <strong>movie</strong>(M,N) plays the movie N times. If N is negative, each
    "play" is once forward and once backward. If N is a vector,
    the first element is the number of times to play the movie and
    the remaining elements comprise a list of frames to play
    in the movie. For example, if M has four frames then
    N = [10 4 4 2 1] plays the movie ten times, and the movie
    consists of frame 4 followed by frame 4 again, followed by
    frame 2 and finally frame 1.
    <strong>movie</strong>(M,N,FPS) plays the movie at FPS frames per second. The
    default if FPS is omitted is 12 frames per second. Machines
    that can't achieve the specified FPS play as fast as they can.
 
    <strong>movie</strong>(H,...) plays the movie in object H, where H is a handle
    to a figure, or an axis.
    <strong>movie</strong>(H,M,N,FPS,LOC) specifies the location to play the movie
    at, relative to the lower-left corner of object H and in
    pixels, regardless of the value of the object's Units property.
    LOC = [X Y unused unused].  LOC is a 4-element position
    vector, of which only the X and Y coordinates are used (the
    movie plays back using the width and height in which it was
    recorded).  All four elements are required, however.

 <strong>fft</strong> Discrete Fourier transform.
    <strong>fft</strong>(X) is the discrete Fourier transform (DFT) of vector X.  For
    matrices, the <strong>fft</strong> operation is applied to each column. For N-D
    arrays, the <strong>fft</strong> operation operates on the first non-singleton
    dimension.
 
    <strong>fft</strong>(X,N) is the N-point <strong>fft</strong>, padded with zeros if X has less
    than N points and truncated if it has more.
 
    <strong>fft</strong>(X,[],DIM) or <strong>fft</strong>(X,N,DIM) applies the <strong>fft</strong> operation across the
    dimension DIM.
    
    For length N input vector x, the DFT is a length N vector X,
    with elements
                     N
       X(k) =       sum  x(n)*exp(-j*2*pi*(k-1)*(n-1)/N), 1 <= k <= N.
                    n=1
    The inverse DFT (computed by IFFT) is given by
                     N
       x(n) = (1/N) sum  X(k)*exp( j*2*pi*(k-1)*(n-1)/N), 1 <= n <= N.
                    k=1
